# PG (Web Payment Gateway) 신청 가이드

**대상**: (주)플레이4 / 일꾼을묻다 운영팀
**목적**: 후보자 가입비 웹 결제 처리를 위한 한국 PG사 가맹 계약
**최종 업데이트**: 2026-05-04

---

## 왜 필요한가

일꾼을묻다는 후보자가 가입비(직급별 19~299만원)를 웹·이메일 링크에서 결제하는 구조다. 이를 위해 **한국 PG사(Payment Gateway) 가맹점 계약**이 필요하다.

기존 GameService 백엔드의 `@jeremybarbet/nest-iap`은 모바일 앱 인앱결제(IAP)용으로, 웹 결제에는 사용할 수 없다.

## 신청 절차 (토스페이먼츠 기준)

### Step 1. 사전 준비 (필요 서류)
- [ ] 사업자등록증 사본 (PDF 또는 이미지)
- [ ] 통장 사본 (정산 대금 입금 계좌)
- [ ] 대표자 신분증 사본
- [ ] 사이트 URL (`https://www.illkkun.cloud`)
- [ ] 판매 상품 설명 (예: "선거 후보자 AI 응답 플랫폼 가입비 — 광역단체장 299만원·기초단체장 129만원·광역의원 39만원·기초의원 19만원")
- [ ] 회사 로고 (PNG·JPG)

### Step 2. 가맹 신청
1. https://www.tosspayments.com/ 접속
2. 우측 상단 **"가맹점 가입"** 클릭
3. 사업자 정보·대표자 정보·통장 정보 입력
4. 사이트 URL 등록 (illkkun.cloud)
5. 판매 상품 카테고리 선택 (가장 가까운 것: "기타 정보 서비스" 또는 "디지털 콘텐츠")
6. 사업자등록증·통장 사본 업로드
7. 약관 동의 후 제출

### Step 3. 심사 대기 (5~7일)
- 토스 측에서 사이트 검수 (illkkun.cloud 실제 운영 중인지)
- 사업자 등록 정보 검증
- 판매 상품의 합법성 확인
- 통상 5~7일 (영업일 기준)

### Step 4. 승인 후 받는 것
- **가맹점 ID** (Merchant ID, MID)
- **시크릿 키** (Secret Key) — 절대 외부 노출 금지
- **클라이언트 키** (Client Key) — 프론트엔드에 사용 가능
- **테스트 키** (Test Key) — 개발용

### Step 5. 연동 작업 (Iran 팀 + 한국 측)
1. **프론트엔드** (illkkun.cloud 결제 페이지):
   - 토스 결제 위젯 JS 임베드 (5~10줄)
   - 가맹점 ID·클라이언트 키만 노출
2. **백엔드** (NestJS):
   - 토스 webhook 수신 엔드포인트 (`POST /api/tosspayments/webhook`)
   - 결제 승인 → DB 저장 → 시트 기록 → 자동 메일 발송
   - 시크릿 키는 `.env`에 보관
3. **테스트**:
   - 테스트 키로 가상 결제 시나리오 검증
   - 결제 성공·실패·취소 케이스 모두 확인
4. **라이브 전환**:
   - 라이브 키로 교체
   - 실제 카드 1건 결제 후 정산 확인 (수수료 차감 후 익영업일 정산)

## 결제 페이지 UI 흐름 (제안)

### 시나리오 A. 별도 결제 페이지 (`payment.html`)

```
1. 후보자가 이메일에서 "결제하기" 링크 클릭
   → https://www.illkkun.cloud/payment.html?candidate=XXX
2. 결제 페이지에 다음 표시:
   - 후보자 정보 (이름·직급·지역구) 자동 표시
   - 가입비 자동 계산 (직급에 따라)
   - [✓] 이용약관 동의 (terms.html 링크)
   - [✓] 개인정보 처리방침 동의 (privacy.html 링크)
   - [결제하기] 버튼
3. 클릭 → 토스 위젯 모달 오픈
   → 카드·계좌이체·간편결제·가상계좌 중 선택
4. 결제 완료 → 토스 webhook → 백엔드 DB 저장 → 자동 메일 (대시보드 ID·PW)
```

### 시나리오 B. Apps Script + 별도 PG 모듈

빠른 운영 방식:
1. 후보자가 메일에서 "결제하기" 클릭
2. 토스의 결제 링크 생성 API로 1회용 결제 URL 생성 → 후보자에게 전송
3. 결제 완료 시 토스 webhook → Apps Script `doPost` 또는 NestJS 엔드포인트
4. 시트에 결제 기록 + 자동 발송

→ **시나리오 B가 빠르지만 시나리오 A가 표준**.

## 비용 구조 (토스페이먼츠 기준)

| 결제 수단 | 수수료율 |
|---|---|
| 신용카드 | 2.9% (협상 시 2.5%까지) |
| 체크카드 | 1.5% |
| 계좌이체 | 1.5% |
| 가상계좌 | 1,000~1,500원/건 (정액) |
| 간편결제 (토스·네이버·카카오) | 2.5~2.9% |

**예시 — 광역단체장 299만원 결제 시**:
- 카드 결제: 299만원 × 2.9% = **86,710원** 수수료
- 정산 금액: 약 290만원

## 운영 시 자주 발생하는 케이스

| 케이스 | 처리 방법 |
|---|---|
| 결제 실패 | 후보자에게 자동 안내 메일 + 재시도 링크 발송 |
| 결제 취소 (구축 전) | 토스 콘솔에서 1클릭 환불 → 시트에 기록 |
| 결제 취소 (구축 후) | 약관에 따라 환불 제한 → 후보자에게 사유 안내 |
| 부분 환불 (회사 사유) | 토스 콘솔에서 부분 환불 → 시트 기록 |
| 정산 지연 | 토스 고객센터 (1599-7977) |
| 카드 분쟁(차지백) | 토스 콘솔에서 거래 증빙 자료 제출 |
