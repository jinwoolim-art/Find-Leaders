1. ROOT CAUSE ANALYSIS
═══════════════════════════════════════════════════════════════

RC1  Exact-string sigungu match in resolveSigungu().
     File: nec-importer.service.ts
     Brittle for typecode 3 (NEC returns sggName=province name) and
     for integrated cities (NEC sggName=청주시, but no local sigungu
     named 청주시 — only 청주시상당구/서원구/흥덕구/청원구).

RC2  No electoral-district entity. NEC's sggName like 종로구제1선거구
     is stored nowhere. After import, emd filter cannot resolve to
     district.

RC3  emdId hardcoded null in processCandidate(). Avatar list's emd
     filter falls back to sigungu/province, leaking neighboring
     districts.

RC4  if (existing) return "skipped" makes prior bad rows permanent.
     New importer fixes do nothing for them.

RC5  Two list endpoints disagree on null semantics.
     - /avatars: strict-with-fallback
     - /candidates: SQL "IS NULL OR =" (fully permissive)
     Same filter returns different sets.

RC6  Seed/dummy NEC rows never reconciled.
     Owner observed Gapyeong-gun dummy rows. Survive every reimport
     because of RC4.

RC7  registDate/status not used for soft-disable.
     NEC sends 사퇴 / 퇴장 / 등록무효 etc., but avatars stay PUBLIC.

User-visible bug = RC1 + RC2 + RC3 (write-time) + RC5 (read-time).
RC4 means fixing write path alone is insufficient — must repair existing.

3.2 NORMALIZER (revised for typecode 4 integrated cities)

  // src/modules/nec-importer/normalize-nec-location.ts
  export async function normalizeNecCandidateLocation(
    candidate: NecCandidate,
    province: Province,
    deps: {
      findDistrict: (q: DistrictQuery) =>
        Promise<ElectoralDistrict | null>;
      findSigungu: (provinceId: string, nameKo: string) =>
        Promise<Sigungu | null>;
    }
  ): Promise<NormalizedLocation> {

    const typecode = candidate.sgTypecode;

    // 3 (governor) and 9 (superintendent) — province-only
    if (typecode === '3' || typecode === '9') {
      return {
        provinceId: province.id,
        sigunguId: null,
        electoralDistrictId: null
      };
    }

    // 4 (mayor/governor) — district may span multiple sigungus
    if (typecode === '4') {
      const district = candidate.sggName
        ? await deps.findDistrict({
            sgTypecode: '4',
            provinceId: province.id,
            sggName: candidate.sggName
          })
        : null;

      // Sigungu resolution priority:
      //   1. wiwName exact match (e.g. "청주시상당구") — most specific
      //   2. sggName exact match (simple case, e.g. "종로구")
      //   3. null
      const sigunguName =
        (candidate.wiwName && candidate.wiwName !== province.nameKo)
          ? candidate.wiwName
        : (candidate.sggName && candidate.sggName !== province.nameKo)
          ? candidate.sggName
        : null;

      const sigungu = sigunguName
        ? await deps.findSigungu(province.id, sigunguName)
        : null;

      return {
        provinceId: province.id,
        sigunguId: sigungu?.id ?? null,
        electoralDistrictId: district?.id ?? null,
      };
    }

    // 5, 7 (council) — district-level
    if (typecode === '5' || typecode === '7') {
      const district = candidate.sggName
        ? await deps.findDistrict({
            sgTypecode: typecode,
            provinceId: province.id,
            sggName: candidate.sggName
          })
        : null;

      // For council races a district has exactly 1 sigungu, take it via
      // electoral_district_sigungu (first row). Fallback to wiwName.
      const sigungu = district
        ? await deps.findFirstSigunguOfDistrict(district.id)
            ?? (candidate.wiwName
                  ? await deps.findSigungu(province.id, candidate.wiwName)
                  : null)
        : (candidate.wiwName
            ? await deps.findSigungu(province.id, candidate.wiwName)
            : null);

      return {
        provinceId: province.id,
        sigunguId: sigungu?.id ?? null,
        electoralDistrictId: district?.id ?? null,
      };
    }

    return {
      provinceId: province.id,
      sigunguId: null,
      electoralDistrictId: null
    };
  }

4.1 SHARED MATCHER

  // src/modules/candidate-filter/public-candidate-filter.ts
  export async function profileMatchesPublicFilter(
    profile: CandidateProfileForFilter,
    filter: PublicListFilter,
    deps: {
      districtCoversEmd:
        (districtId: string, emdId: string) => Promise<boolean>;
      districtCoversSigungu:
        (districtId: string, sigunguId: string) => Promise<boolean>;
    }
  ): Promise<boolean> {
    if (profile.electionTypeId !== filter.electionTypeId) return false;
    if (profile.provinceId !== filter.provinceId) return false;

    if (filter.emdId) {
      if (!profile.electoralDistrictId) return false;
      return await deps.districtCoversEmd(
        profile.electoralDistrictId, filter.emdId
      );
    }

    if (filter.sigunguId) {
      // Path A: district coverage — required for integrated cities
      //         where profile.sigunguId is a sub-sigungu of a wider district.
      if (profile.electoralDistrictId) {
        const covered = await deps.districtCoversSigungu(
          profile.electoralDistrictId, filter.sigunguId
        );
        if (covered) return true;
      }
      // Path B: direct match — for manual candidates without district
      return profile.sigunguId === filter.sigunguId;
    }

    return true;
  }

5.1 M4 — Excel ingestion

  Reads ai-avatar-core/scripts/seed-data/election-2026-06-03/*.xlsx
  Idempotent.

  Stable filenames:
    sd-uihoe.xlsx       (sgTypecode=5, 시·도의회의원선거)
    gusigun-uihoe.xlsx  (sgTypecode=7, 구·시·군의회의원선거)
    gusigun-jang.xlsx   (sgTypecode=4, 구·시·군의_장선거)

  Steps per file:
    1. Each sheet name = province nameKo → map to province.id.
    2. Header row is row 4. Columns:
         구시군명 | 선거구명 (sggName) | 정수 | 읍면동명 (CSV emds)
    3. For each row:
       a. Normalize sigunguName via normalizeKoreanName().
       b. Resolve sigunguId by (provinceId, normalized nameKo).
       c. Upsert electoral_district keyed by
            (sgTypecode, provinceId, sggName).
       d. Upsert electoral_district_sigungu keyed by
            (electoralDistrictId, sigunguId).
       e. For integrated city typecode 4 (e.g. 청주시), some Excel rows
          may have districtName blank (continuation of previous district).
          Use the last-seen districtName until a new one appears.
          The result: one district 청주시 linked to 4 sigungus.
       f. Split 읍면동명 by ',' and normalize each.
       g. Resolve emd by (sigunguId, normalized emd.nameKo).
       h. Upsert electoral_district_emd keyed by
            (electoralDistrictId, emdId).
       i. If emd not found → nec_import_report outcome=unmatched-emd.

  Wrap per sheet in a transaction. Idempotent on rerun.

5.2 M5 — NEC repair (scoped to user.necHuboid IS NOT NULL)

  FOR EACH user u WHERE u.necHuboid IS NOT NULL:
    cp = profile for u
    IF cp.electoralDistrictId IS NULL:
      district = find_district by
        (sgTypecode of u.electionType, cp.provinceId, cp.sggNameRaw)
      IF district found:
        cp.electoralDistrictId = district.id
        IF cp.sigunguId IS NULL:
          cp.sigunguId = first_sigungu_of(district) or null
      ELSE:
        report unmatched-district
    IF cp.sigunguId IS NULL AND district found AND
       district has exactly 1 covered sigungu:
      cp.sigunguId = that sigungu

    Never touch: symbolNumber, slogan, greeting, privateAccessKey,
                 avatar visibility/content.

  Rows where sggNameRaw is still null (imported before §2 fields existed)
  are re-derived on the NEXT importer cycle, now that the new
  importer §3.3 writes raw fields every upsert. M5 does only what it
  can with current state.
