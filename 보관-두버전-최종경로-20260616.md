## ④ 영상 자동화 연결 — "나중에 켤 작업" (variation/실서비스 시)

HTML 버전엔 **연동 코드만 보관**돼 있고 스위치가 꺼져 있음(`AVATAR_VIDEO_BACKEND_ENABLED = false`).
지금은 기존 하드코딩/이미지 폴백만 동작. API 연결 시 아래를 적용하고 스위치를 켜면 즉시 작동.

**파이프라인 구조** (이미 완성):
이미지 업로드(host-panel) → "동영상 생성" 버튼(수동, 토큰 1000·쿨다운 120s) → core `POST /candidate/avatars/:id/request-avatar-video` → 외부 railway 서버가 영상 생성 → webhook 2단계 → `AVATAR_VIDEO` asset(READY) 저장

**variation 때 켤 작업 5가지:**
1. 백엔드 `ai-avatar-core .../avatar.service.ts:200` `toPublicAvatarResponse` 필터에 `'AVATAR_VIDEO'` 추가 (현재 PROFILE_IMAGE/VOICE_SAMPLE/MOTION_CLIP만)
2. 거주지 쿼리(electionTypeId·provinceId) 없이 영상 URL만 받는 공개 경로 신설 (예: `GET /avatars/:id/video`)
3. 게스트 권한: `auth.service.ts` `isAllowedGuestAuthenticatedRequest`에 위 경로 추가
4. v3 스위치 `AVATAR_VIDEO_BACKEND_ENABLED = true`
5. DB에 `AVATAR_VIDEO` asset 생성 후 e2e 검증 (현재 0개)

> 관련 메모리: `project_avatar_html_video_link`

## 보관 개념 요약
- **Unity = 완료**, **HTML = 영상 자동화 연결 코드까지 보관**
- 둘 다 **API 미연결 상태** — 백엔드 꺼져 있어도 폴백으로 동작
- HTML 배포본은 백엔드 다운 시 자동 폴백 (`CITIZEN_API_BASE` localhost/sandbox 자동 분기)
- 필요 시 API만 연결하면 즉시 풀 기능 — variation 제작 베이스
